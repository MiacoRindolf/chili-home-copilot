- Contract coverage deduplicates exception suffixes, maps each unique failed identity, and cannot pass by count.
- Contract owner mappings replace unowned draft file guesses; imported feedback context no longer grants edits.
- Unknown or advisory plan actions are fail-closed; only explicit `modify` and `create` actions mutate.
- Base multi-file repair keeps the proven per-file adapter with shared plan context; compact 14B escalation can use
  one atomic coordinated bundle.
- Repeated plans against the same normalized failure signature do not fan out another edit attempt.
- Changed SQLite scripts materialize in supplied order inside an isolated in-memory database with attachment and
  extension loading blocked.
- Context-only and inferred evidence can rank hypotheses but cannot add causal support. Prompt taxonomy is
  advisory unless backed by graph-linked, isolated, or explicit owner-aligned evidence.
