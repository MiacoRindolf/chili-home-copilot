### Step C — process-level circuit breaker

Passive counter. Module-level state guarded by `_breaker_lock`. Three states: `CLOSED` → `OPEN` → `HALF_OPEN` → `CLOSED|OPEN`.

Constants (named, no magic):
- `_BREAKER_CONSECUTIVE_FAILURE_THRESHOLD = 10` — number of consecutive failures to trip OPEN.
- `_BREAKER_HALF_OPEN_TTL_S = 60.0` — TTL before allowing a probe call.

Public helpers (private to the module):
- `_breaker_should_short_circuit() -> bool` — call before each yfinance hit. Returns `True` only when `OPEN` and TTL not yet elapsed; transitions OPEN → HALF_OPEN in place when TTL elapses (so the next call probes through).
- `_breaker_on_success()` — resets counter, transitions to `CLOSED`, logs `[yf_breaker] CLOSED: success after <prev_state>` only on transition.
- `_breaker_on_failure()` — increments counter, trips OPEN at threshold or on HALF_OPEN failure, logs `[yf_breaker] OPEN: <N> consecutive upstream failures` only on transition.
- `_reset_breaker_for_tests()` — test-only reset.

Wiring in entry points:
- Cache hits / `_is_dead` short-circuits → no breaker activity.
- `_breaker_should_short_circuit()` runs before `acquire()` so a tripped breaker doesn't even spend a rate-limit token.
- Successful response → `_breaker_on_success()`.
- Exception, or empty DataFrame/None on a stock → `_breaker_on_failure()`.
- Empty DataFrame on crypto → no breaker tick (per the brief; crypto-empty is non-signal).

## Surprises / deviations

### 1. The pre-fix slope was 3.5× the brief's reference (178/min vs 50/min)
The brief's mem_watcher tick at 06:19:26 UTC observed ~50/min growth. By the time I started the diagnostic at ~07:16 UTC, the rate had climbed to ~178/min. Either:
- The leak rate is non-linear (yfinance's internal cache/state grows over time, making each failed call leak more), OR
- A traffic spike during the observation window (the autotrader's pattern engine sweeping more tickers)

Doesn't change the fix shape — both layers (shared session + breaker) cap the per-call leak regardless of arrival rate.

### 2. yfinance 1.3.0 docstring caveat is outdated
The yf_session.py:1-16 docstring originally claimed "modern yfinance >=0.2.40 ... rejects injected sessions." Verified by `inspect.signature(yf.Ticker.__init__)` that `session=` IS supported on 1.3.0. Updated docstring; the older caveat referred specifically to `requests-cache` wrapping, not vanilla `curl_cffi.Session` injection.

### 3. WinError 10055 in the test suite confirms the leak's host impact
The 1 setup-error in `test_provider_selection` is exactly the symptom the rate-limiter docstring warns about ("WinError 10055 socket-pool exhaustion that blocked every subsequent socket.connect()"). The leak has accumulated enough handles in the non-paged kernel pool to fail new socket binds. After 30 min of soak with the fix landed, this should clear as the OS reclaims handles.

### 4. f-leak-2 already lifted mem_watcher into chili
Pre-existing post-f-leak-2 state. Just consumed; no additional plumbing needed.

## Open questions for Cowork

1. **Threshold tuning** — `N=10` and `TTL=60s` seed values. Empirical signal during the verification window will say whether these need to move. Recommend tracking `[yf_breaker] OPEN/CLOSED` flap rate over 24h post-deploy. If the breaker oscillates rapidly (open → closed → open within minutes), bump TTL or threshold; if it never trips (yfinance recovers), the seeds are fine.
2. **Reset-on-cache-hit semantics** — current design: cache hits don't tick the breaker either way. Alternative: cache hits could count as success (resetting the failure counter), making the breaker more lenient when the cache has lots of hits. Not implemented; surfacing as a tuning option.
3. **Curl_cffi impersonation profile** — `impersonate="chrome"` is the default. Yahoo's CDN occasionally bumps allowed profiles. If post-fix sees a `403` storm where pre-fix saw connection failures, surface and consider rotating to `safari` or `chrome131`.
4. **Scheduler-worker drop-or-not** — surfaces as part of the verification observation. If scheduler's `_make_invoke_excepthook` drops in parallel with chili's, one fix covers both processes; if not, separate brief.
