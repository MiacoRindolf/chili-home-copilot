## Cross-phase observations

1. **Source-text guards as a precedent**. Several phases shipped tests that pin source-level invariants (wiring, log lines, import patterns, dispatch order) instead of full DB-integration tests. At <1s per test file vs 7-10 min for handler-touching DB tests under the per-test truncate cycle, the guards catch the same regression class (accidental future deletion of wiring) at a tiny fraction of the runtime cost. Worth flagging as a Cookbook entry.

2. **The depth-5 absolute-imports rule is now load-bearing**. Phases 3, 4, 5, 6, 9 all created modules in `cron_jobs/` or `handlers/` (depth 5 from `app`). Each module uses absolute imports per the f-handler-pattern-stats finding. The pattern is now: in any `app/services/trading/X/Y/Z.py` module, prefer `from app.X import Y` over `from ....X import Y`. Documented in module docstrings; tests pin via anti-pattern grep checks.

3. **The brain_work handler tree is now 9 handlers strong**: mine, cpcv_gate, promote, demote, regime_ledger, pattern_stats, breakout_outcomes (Phase 3), live_drift (Phase 6), execution_robustness (Phase 6). All 9 use the same shape (fresh SessionLocal, absolute imports, try/except/finally). PHASE2_HANDLER_BACKLOG.md should be updated to reflect this run.

4. **Cron jobs are emerging as the right pattern for cycle-step replacement.** Phases 4, 5, 9 all ship cron-driven cleanup/observability work. Distinction from event-handlers: cron when the natural trigger is "every N hours", event-handler when the trigger is "this thing happened". `cron_jobs/` package is now the canonical home for the former.

5. **Dispatcher's close-event branch is getting busy**. After Phase 6, the trade-close fanout calls 5 handlers in sequence: `pattern_stats → demote → live_drift → execution_robustness → regime_ledger`. If any of them slows down, the entire round slows. Surface as a watch-item: if post-deploy logs show >30s per close-event dispatch, consider parallelising via threading or moving slower handlers to per-handler dedicated queues.

## Cookbook updates from this run

1. **Source-text test guards** are a legitimate test category alongside DB-integration tests. Use them when the regression class is "accidental future deletion of wiring." Sub-1s per test file. Document the import / log-line / dispatch-order invariant in plain English in the test docstring.

2. **Anti-pattern grep checks** in tests catch the f-handler-pattern-stats import-bug class. `assert "from ....db" not in src` for any new module at depth-5+. Cheap, catches the entire class.

3. **`cron_jobs/` vs `brain_work/handlers/`**: cron when the natural trigger is "every N hours" (broad-market state, periodic cleanup, observability snapshots); event-handler when "this thing happened" (per-trade-close, per-resolved-alert).

4. **Per-event try/except inside emit batches**: when emitting N events from a sweep, wrap each emit in try/except so one broken event can't poison the rest. Pattern shipped in Phases 3 + 4.

5. **Defer the disk-retention policy** when shipping a new logging/snapshot path. Surface in the CC report; operator decides whether to add rotation now or via a follow-up.
