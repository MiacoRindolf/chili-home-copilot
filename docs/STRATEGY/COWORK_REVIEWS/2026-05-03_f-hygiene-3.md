## Findings

### The 49-min-hold → horizon=3600 mapping is the more strategically interesting bug

`min(HORIZONS_S, key=...)` snaps the realized-validation residual to the closest of {1, 5, 30, 60, 300, 1800, 3600, 14400}s. Pullback round-trips average 49 minutes (= 2940s). Closest is 3600. **All validations land at horizon=3600.** Shorter horizons (1, 5, 30, 60, 300, 1800) get zero realized-residual signal regardless of how many exits accumulate.

Implications:
- The decay-miner's per-bucket-per-horizon mean reflects forward-return observations only. The realized-residual signal — which is what F8a-evaluation-rerun argued was the more strategically informative lens — accumulates at a single horizon, not all 8.
- F8a-evaluation-rerun's "validation_count = 7" was actually concentrated, not spread. Once the UPSERT lands cold-cell rows, they'll all be at horizon=3600.
- For F9, signal classes with shorter expected hold times will have validations land at shorter horizons. The 8-horizon framework is fine; the realized-residual coverage is just per-signal-class-narrow.

This isn't a bug — it's a structural property of the closest-horizon match. **But it changes how f8a-evaluation-rerun-2 should interpret realized-residual data.** The brief for B should call this out so Claude Code uses horizon=3600 specifically for pullback realized-validation analysis.

### The cold-start backfill variance artifact is calibration-relevant but not blocking

`decay_miner.py:846` aggregates dup alerts as separate observations during cold-start backfill. Five dups → 5 forward_return rows with identical (numerator, denominator) → identical Welford increments → mean unchanged but VAR_POP = 0 (or near-zero). Calibration's stderr-based gates (`gate_negative_edge_excluded`, `gate_tradeable`) get input where stderr is artificially small, biasing toward false confidence in the bucket cell.

For F8a evaluation specifically, this means **suggestive-tier cells with low stderr might be artifacts** of duplicate observations rather than genuinely tight measurements. The F8a-evaluation-rerun report noted SOL-USD's `stderr=0` cells as a quantization artifact; this is a related-but-different artifact (dup-aggregation, not tick quantization).

**Not in scope for f-hygiene-3, surfaced for f8a-evaluation-rerun-2.** Should not block forward motion; should be an interpretation caveat in the next evaluation.

### F-hygiene-3 is the test of "queue an analysis-grade hygiene task that maybe doesn't have work to do"

Subtask 2 was branch C. That's exactly what the brief called out as a possible outcome ("If Subtask 2 finds zero sites needing fix, that means F-hygiene-2.1 already covered everything"). Branch C is success — it means the prior hygiene's work was complete, not that this hygiene wasted time.
