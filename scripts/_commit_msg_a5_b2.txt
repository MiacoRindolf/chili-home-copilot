fix(audit-2026-04-29): A-5 broker-qty clamp + B-2 avg_return_pct backfill

A-5 (15 "Not enough shares to sell" rejections / 24h):
  robinhood_exit_execution.py now resolves the broker's actual position
  quantity via get_positions() before submitting a sell. If the broker
  holds less than the local Trade.quantity (partial fill, manual sale,
  stale broker_sync), the sell quantity is clamped to broker truth. If
  the broker holds zero, the trade is treated as no_position. If the
  broker quantity is unknown (API failure), the exit is deferred per
  the no-hardcoded-fallback rule -- never silently widen.

  Crypto path is unchanged (positions API is stocks-only and crypto
  uses a different endpoint).

  patterns whose only evidence is in trading_pattern_trades (backtest
  replays) -- about 188 patterns. Aggregates use outcome_return_pct
  (percent, post-mig-208 clamp) and label_win. Min sample threshold
  reuses chili_realized_ev_min_trades (5) so we do not promote
  fluke-driven stats. Idempotent: only writes when avg_return_pct IS
  NULL, so it does not overwrite the realized values mig 207 already

  concern.
