fix(prescreen): round-16 brain-worker idle-tx leak on prescreen reads

User-directed audit of brain-worker learning cycle. Found 20 idle-in-
transaction sessions on `chili-brain-worker`, oldest 25 min old, all
on `SELECT trading_prescreen_candidates ... ORDER BY ticker_norm`.

Same bug class as FIX 46 (scanner session leak):
SQLAlchemy `session.close()` does NOT end an implicit read transaction.
A SELECT-only path that doesn't commit/rollback leaves the session
in 'idle in transaction' state until something explicitly clears it.
Concurrent worker threads stack up these orphan sessions, each
holding a read snapshot on `trading_prescreen_candidates` and a
connection from the pool.

Fix: explicit `db.rollback()` after the SELECT in
`load_active_global_candidate_tickers`. The read has no writes to
flush, so rollback is the cheapest release of locks + connection
without forcing every caller to know about transaction state.

Also killed the 16 leaked sessions live (immediate relief).
